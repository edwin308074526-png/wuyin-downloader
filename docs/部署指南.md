## 🚀 后端部署方案

### 方案一：部署到云服务器（推荐）

#### 1. 环境要求
- Node.js >= 16.0.0
- npm 或 yarn
- 1GB+ 内存
- 公网IP或域名

#### 2. 部署步骤

```bash
# 1. 进入后端目录
cd video-downloader/backend

# 2. 安装依赖
npm install

# 3. 启动服务
npm start

# 4. 后台运行（使用pm2）
npm install -g pm2
pm2 start server.js --name video-api
pm2 save
pm2 startup
```

#### 3. 使用Docker部署

创建 `Dockerfile`：
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

创建 `docker-compose.yml`：
```yaml
version: '3.8'
services:
  api:
    build: .
    ports:
      - "3000:3000"
    restart: always
    environment:
      - NODE_ENV=production
      - PORT=3000
```

启动：
```bash
docker-compose up -d
```

### 方案二：部署到Serverless

#### 部署到阿里云函数计算

1. 安装函数计算CLI：
```bash
npm install @serverless/devs -g
```

2. 创建 `s.yaml` 配置

3. 部署：
```bash
s deploy
```

#### 部署到腾讯云SCF

1. 打包代码：
```bash
cd backend
npm install
zip -r function.zip .
```

2. 上传至腾讯云函数

### 方案三：使用第三方API服务

如果自建服务有困难，可以使用以下方案：

1. **使用已有解析API**
   - 缺点：不稳定，可能收费

2. **使用Railway/Render等平台**
   - 提供免费额度
   - 自动 HTTPS

## 🌐 小程序端配置

### 1. 配置合法域名

在小程序根目录的 `app.js` 中修改API地址：

```javascript
globalData: {
  apiBaseUrl: 'https://your-api-domain.com', // 替换为你的API地址
}
```

### 2. 在微信公众平台配置

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「开发」→「开发管理」→「开发设置」
3. 配置「服务器域名」：
   - request合法域名：你的API域名
   - downloadFile合法域名：同上
   - uploadFile合法域名：同上（可选）

### 3. 上传和发布

1. 使用微信开发者工具打开 `miniprogram` 目录
2. 修改 `project.config.json` 中的 `appid`：
   ```json
   "appid": "wxf5ec732be56b6766"
   ```
3. 点击「上传」按钮
4. 在公众平台提交审核

## ⚠️ 注意事项

### 1. 域名备案
- request合法域名必须使用已备案的域名
- 可使用免费域名（如Freenom）或购买阿里云/腾讯云域名

### 2. HTTPS证书
- 必须使用HTTPS
- 可使用Let's Encrypt免费证书
- 阿里云/腾讯云提供免费证书

### 3. 视频解析限制
- 各平台可能随时更新反爬策略
- 可能需要定期更新解析逻辑
- 建议添加错误处理和重试机制

## 🔧 常见问题

### Q: 解析失败怎么办？
A: 
1. 检查网络连接
2. 确认API服务是否正常运行
3. 检查链接是否正确
4. 可能是平台更新了反爬策略，需要更新解析代码

### Q: 如何添加新平台？
A:
1. 在 `services/` 目录创建新的解析服务文件
2. 参考现有服务实现 `parse(url)` 函数
3. 在 `server.js` 中引入并添加路由

### Q: 如何提高解析成功率？
A:
1. 使用真实浏览器Headers
2. 添加重试机制
3. 维护多个备用解析方法
4. 考虑使用第三方解析服务作为补充
